Selected Experiments

Can dependency injection be made visible without runtime instrumentation?

Large .NET solutions obscure DI configuration across multiple files and assemblies, making lifetime conflicts hard to detect.

Static analysis with Roslyn reveals service lifetime conflicts, dependency chains, and injection patterns before runtime—turning invisible architecture into navigable structure.

TypeScript Roslyn VS Code API
di-navigator

How do you prevent configuration drift across Kubernetes environments?

When development, staging, and production Helm deployments diverge silently, production incidents become detective work.

Side-by-side field-level diffs with visual highlighting make drift immediately obvious—comparison becomes the interface.

TypeScript Helm CLI Chart.js
chart-profile-visualizer

What if serialization formats prioritized human comprehension?

JSON optimizes for machines, but engineers spend hours scanning deeply nested structures for debugging.

Schema-first formats with visual compression reduce cognitive load—structure should aid scanning, not hinder it.

C# .NET 10
PloonNet

Can observability configuration be validated before deployment?

Grafana Alloy configuration syntax errors discovered at runtime break telemetry pipelines in production.

Real-time diagnostics with language server protocols prevent broken observability—validation is part of authoring, not deployment.

TypeScript VS Code LSP
grafana-alloy-toolkit

How do you sync database schemas across distributed teams?

PostgreSQL schema changes cause deployment friction when scattered across migration files and documentation.

Extension-based workflows reduce context switching—database operations should live in the editor, not separate tools.
